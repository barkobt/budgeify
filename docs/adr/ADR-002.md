# ADR-002: ESLint Configuration with TypeScript Strict Rules

**Status:** Accepted
**Date:** 2026-02-06
**Context:** No ESLint was configured. `next lint` prompted setup interactively. No automated code quality enforcement.

**Decision:** Configure ESLint with `next/core-web-vitals` + `@typescript-eslint/recommended`. Add `no-console` as warning, `no-explicit-any` as error. Add `npm run lint` and `npm run typecheck` to quality gates.

**Consequences:**
- Consistent code quality enforcement
- Catches `any` usage and unused variables
- Warns on stray console statements
