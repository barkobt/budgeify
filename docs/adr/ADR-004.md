# ADR-004: Security Headers & Production Hardening

**Status:** Accepted
**Date:** 2026-02-06
**Context:** No security headers configured. No CI pipeline. No vercel.json for deployment optimization.

**Decision:**
1. Add security headers in both `next.config.mjs` (SSR) and `vercel.json` (edge):
   - X-Content-Type-Options: nosniff
   - X-Frame-Options: DENY
   - X-XSS-Protection: 1; mode=block
   - Referrer-Policy: strict-origin-when-cross-origin
   - Permissions-Policy: camera=(), microphone=(), geolocation=()
   - Strict-Transport-Security: max-age=31536000; includeSubDomains

2. GitHub Actions CI pipeline runs typecheck, lint, and build on every push/PR to main.

3. vercel.json adds long-lived cache headers for static assets (icons).

**Consequences:**
- Defense-in-depth against common web attacks
- Automated quality enforcement on all commits
- Faster icon loading via immutable caching
